# 百度竞价落地页 URL 匹配助手

## 这个 Skill 做什么

把百度竞价账户里的关键词批量匹配到对应的落地页 URL。解决多页面、多域名、账户结构不统一时的 URL 分配效率问题。

核心是一个本地离线 HTML 工具（`tool.html`），双击即可在浏览器运行，所有数据存在浏览器本地，不上传任何服务器。

## 唯一执行流程

严格按以下顺序执行，不要跳步：

### 1. 识别用户意图

用户说出触发词或表达以下意图之一时启动本 Skill：
- "帮我匹配落地页 URL"
- "关键词对应的 URL 怎么分配"
- "批量给关键词配 URL"
- "我有一批关键词要分到不同落地页"
- "把这张 URL 规则表/关键词表处理一下"

### 2. 判断用户当前需求类型

按用户已有材料判断走哪条路径：

| 用户提供的材料 | 执行路径 |
|---|---|
| 只有关键词或 Excel 关键词表，想生成 URL | **路径 A：用工具匹配** |
| 有现成的关键词表 CSV/Excel，想从真实数据反向构建规则 | **路径 B：基于真实数据构建规则** |
| 想调整/查看/导入导出已有规则 | **路径 C：规则维护** |
| 只是问工具怎么用 | **路径 D：使用说明** |

### 3. 按路径执行

#### 路径 A：用工具匹配（最常见）

1. 打开 `tool.html`（用 PowerShell `Start-Process` 或 Bash `start` 命令在默认浏览器打开）。
2. 引导用户：
   - 先到「域名管理」Tab 添加落地页域名
   - 再到「规则配置」Tab 添加匹配规则（按优先级从上到下）
     - 少量规则：点「+ 添加规则」逐条建
     - 批量规则：用「批量粘贴建规则」卡片，从 Excel 复制三列（计划/单元/URL）粘贴即可批量生成
     - 规则多时：用搜索框过滤，勾选多条规则可批量启用/禁用/删除/复制
   - 最后到「匹配执行」Tab 粘贴关键词执行匹配
3. 用户从 Excel 复制关键词粘贴时，确认输入格式：
   - 仅关键词
   - 计划+单元+关键词（三列 Tab 分隔，最常用）
   - 计划+单元+关键词+类别（四列）
4. 执行后结果可一键复制回 Excel，或导出 CSV。

#### 路径 B：基于真实数据构建规则

当用户给出真实的百度竞价关键词导出 CSV/Excel 时：

1. **识别文件编码**：百度后台导出通常是 UTF-16 LE + BOM。用 Python 读取时先去 BOM 再 decode。
2. **识别表头字段**：百度标准表头 20 列，关键字段是 `推广计划名称`、`推广单元名称`、`关键词名称`、`计算机端落地页链接`、`计算机监控后缀`。
3. **分析账户结构**：统计域名数、计划数、单元数、URL 去重数、单元→URL 映射关系。
4. **区分两种 URL 规律**：
   - **有规律单元**（同一单元下主 URL 占比 ≥ 80%）：生成单元级规则
   - **无规律单元**（主 URL 占比 < 80%，通常是真实词指向不同详情页）：无法用单元规则匹配，需导出关键词→URL 映射表单独查
5. **处理同名单元**：若同一单元名出现在多个计划下且指向不同 URL，使用 `plan_unit_exact` 类型（value 格式 `计划|||单元`）。
6. **生成 JSON 规则文件**：结构为 `{domains, rules, defaultUrl, lastResults}`，可由工具「导入规则」一键加载。
7. **全量验证**：用生成规则回跑全部关键词，输出准确率。有规则覆盖部分准确率应为 100%。
8. **输出物**：
   - `<账户名>_匹配规则配置.json`（导入工具用）
   - 无规律单元的关键词→URL 映射表 CSV（单独查表用）

#### 路径 C：规则维护

- 导入规则：工具顶部「导入规则」选 JSON 文件
- 导出规则：工具顶部「导出规则」存 JSON 备份
- 清空数据：工具顶部「清空数据」（会清除所有规则和域名，谨慎）

#### 路径 D：使用说明

直接引用 `tool.html` 内置的「使用说明」Tab 内容，包括：
- 匹配类型说明（10 种）
- 变量替换规则（`{{kw}}` `{{plan}}` `{{unit}}` `{{category}}` 双花括号，避免和百度原生通配符 `{keyword}` 冲突）
- 输入格式说明
- 典型场景示例

## 匹配类型速查

| 类型 | value 格式 | 用途 |
|---|---|---|
| keyword_exact | 关键词文本 | 关键词完全相等 |
| keyword_contains | 子串 | 关键词包含子串 |
| keyword_regex | 正则 | 关键词匹配正则 |
| unit_exact | 单元名 | 单元名完全相等 |
| unit_contains | 子串 | 单元名包含子串 |
| plan_exact | 计划名 | 计划名完全相等 |
| plan_contains | 子串 | 计划名包含子串 |
| plan_unit_exact | `计划\|\|\|单元` | 计划+单元同时精确匹配（解决同名单元跨计划） |
| category_exact | 类别名 | 业务类别完全相等 |
| category_contains | 子串 | 业务类别包含子串 |

规则按列表顺序从上到下匹配，命中第一条即停止。都未命中走默认 URL。

## 变量替换

在路径或查询参数中使用双花括号变量，匹配时自动替换：

| 变量 | 含义 | 编码 |
|---|---|---|
| `{{kw}}` | 关键词 | URL 编码 |
| `{{kw_raw}}` | 关键词原始值 | 不编码 |
| `{{plan}}` / `{{plan_raw}}` | 计划名 | 编码 / 原始 |
| `{{unit}}` / `{{unit_raw}}` | 单元名 | 编码 / 原始 |
| `{{category}}` / `{{category_raw}}` | 业务类别 | 编码 / 原始 |

**重要**：百度竞价原生通配符 `{keyword}` `{planid}` `{userid}` 等单花括号不会被替换，原样保留。从真实 URL 提取规则时，这些通配符在 URL 中通常是 URL 编码形式（`%7Buserid%7D`），保留原始编码即可。

## 关键约束

- **本地离线**：工具是纯 HTML+CSS+JS 单文件，不依赖网络，数据存浏览器 localStorage。
- **不修改用户原始数据**：分析真实 CSV 时只读取，不写回原文件。
- **规则文件不依赖路径**：JSON 规则文件可在任意机器导入，不绑定绝对路径。
- **真实词单元不可强行规则化**：若同一单元下每个关键词指向不同详情页（如 `/yuanqu/detail/{id}`），不能用单元规则覆盖，必须导出映射表。

## 启动工具

工具文件：`tool.html`（与本 SKILL.md 同目录）

启动命令（PowerShell）：
```powershell
Start-Process "$env:USERPROFILE\.workbuddy\skills\baidu-landing-url-matcher\tool.html"
```

启动命令（Bash）：
```bash
start "" "$USERPROFILE/.workbuddy/skills/baidu-landing-url-matcher/tool.html"
```

也可以直接把 `tool.html` 复制到桌面双击使用。

## 输出落盘

- 生成的规则 JSON 写到用户工作目录（不是 skill 目录），文件名格式：`<账户名>_匹配规则配置.json`
- 关键词映射表 CSV 写到用户工作目录：`<账户名>_关键词URL映射表.csv`
- 不要把用户数据写到 skill 目录内部
